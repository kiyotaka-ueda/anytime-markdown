# Markdown設計書作成に必要な機能の検討

日付: 2026-03-03
ステータス: ブレスト完了

## 背景

anytime-markdown を使ってソフトウェア設計書（基本設計・詳細設計・API仕様等）を作成する場合に、現状で不足している機能を洗い出す。

## 現状で既に使える機能

| カテゴリ | 機能 | 評価 |
|---------|------|------|
| 図表 | Mermaid（フローチャート, シーケンス, クラス, ER, ステート, ガント等） | 充実 |
| 図表 | PlantUML（ユースケース, デプロイメント, コンポーネント等） | 充実 |
| 図表 | テーブル（行列操作, 配置変更, 全画面編集） | 充実 |
| 数式 | KaTeX（インライン + ブロック） | 十分 |
| 構成 | 見出し H1〜H5 + TOC自動生成 + アウトラインパネル | 十分 |
| コード | シンタックスハイライト付きコードブロック | 十分 |
| 折りたたみ | Details（`<details>/<summary>`） | 十分 |
| HTML | カスタムHTMLプレビュー | 十分 |
| 比較 | Diff/マージビュー | 十分 |
| 検索 | 検索・置換（正規表現対応） | 十分 |
| 出力 | PDF エクスポート（バグあり） | 要改善 |
| テンプレート | 4種（サンプル, 会議, README, ブログ） | 不足 |

## 不足している機能候補

### A. レビュー・コメント機能（ユーザー要望: 高）

#### A1. インラインコメント / 注釈
- テキスト選択 → コメント追加
- コメントのスレッド表示（返信可能）
- 解決済み / 未解決のステータス管理
- コメントパネルで一覧表示
- Markdown 出力時は HTML コメント `<!-- comment -->` として保存

**実装規模**: 大（新規 Extension + UI コンポーネント）
**優先度**: ★★★

#### A2. 変更提案（Suggestion）
- レビュアーが「この部分をこう変えるべき」と提案
- Accept / Reject で反映
- 既存の Diff ビューとの連携

**実装規模**: 大
**優先度**: ★★☆

### B. ドキュメント構造・メタデータ

#### B1. YAML フロントマター対応
- ドキュメント先頭に `---` で囲んだメタデータ
- タイトル、バージョン、著者、ステータス、日付
- エディタ上部にバナー/ヘッダーとして表示
- テンプレートに組み込み可能

```yaml
---
title: API設計書
version: 1.2
author: 田中太郎
status: レビュー中
date: 2026-03-03
---
```

**実装規模**: 中
**優先度**: ★★★

#### B2. セクション自動番号
- 見出しに自動的に「1.」「1.1」「1.1.1」形式の番号を付与
- 設定で ON/OFF 切り替え
- TOC にも番号を反映

**実装規模**: 小〜中
**優先度**: ★★☆

#### B3. 図表の番号・キャプション
- 「図1: システム構成図」「表2: API一覧」のように自動番号
- 本文中から `[図1]` のように参照リンク生成

**実装規模**: 中
**優先度**: ★★☆

### C. 注釈・補足表現

#### C1. Admonition / Callout ブロック
- Note, Warning, Tip, Important, Caution 等の装飾ブロック
- 設計上の注意点、制約事項、決定理由の強調に有用
- GitHub / Obsidian 互換の `> [!NOTE]` 記法

```markdown
> [!WARNING]
> この API は後方互換性を保証しません。
```

**実装規模**: 中
**優先度**: ★★★

#### C2. 脚注（Footnotes）
- 参考仕様、外部ドキュメントへの参照
- `[^1]` 記法で本文中にマーク、ページ下部に脚注表示

**実装規模**: 小〜中
**優先度**: ★★☆

### D. テンプレート拡充

#### D1. 設計書テンプレート追加
- 基本設計書（概要, 要件, アーキテクチャ, DB設計, 画面設計）
- 詳細設計書（クラス設計, シーケンス, API仕様）
- API仕様書（エンドポイント一覧, リクエスト/レスポンス例）
- ADR（Architecture Decision Record: 決定事項, コンテキスト, 選択肢, 結果）
- DB設計書（テーブル定義, ER図, インデックス設計）

**実装規模**: 小（テンプレートJSONの追加のみ）
**優先度**: ★★★

### E. エクスポート強化

#### E1. PDF エクスポートのバグ修正
- Mermaid レンダリング後の「準備中」停止バグの修正
- タイムアウト機構の追加

**実装規模**: 小
**優先度**: ★★★（前提条件）

#### E2. PDF ヘッダー/フッター
- ドキュメントタイトル、ページ番号、日付をヘッダー/フッターに表示
- フロントマターから自動取得

**実装規模**: 小〜中
**優先度**: ★★☆

#### E3. HTML 単体エクスポート
- スタイル埋め込みの単体 HTML ファイル出力
- メール添付やブラウザ表示用

**実装規模**: 中
**優先度**: ★☆☆

#### E4. DOCX エクスポート
- Word 形式での出力（顧客提出用）
- pandoc 連携 or ライブラリ（docx）使用

**実装規模**: 大
**優先度**: ★☆☆

### F. その他

#### F1. 変数・マクロ
- `{{project_name}}` のような変数を定義し、文書全体で再利用
- フロントマターで定義 → 本文中で展開

**実装規模**: 中
**優先度**: ★☆☆

#### F2. ファイル分割・インクルード
- 大規模設計書を複数ファイルに分割
- `!include path/to/section.md` で結合

**実装規模**: 大
**優先度**: ★☆☆
