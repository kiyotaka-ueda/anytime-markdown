# モバイルアプリの言語切替（クライアントサイド i18n）

## 意図

Capacitor 静的エクスポートでは `cookies()` が使えず、ビルド時にロケールが ja に固定される。現在の言語切替（cookie + `window.location.reload()`）はモバイルアプリで機能しない。

クライアントサイドで動的にロケールを切り替える仕組みを導入し、モバイル・Web 両方で言語切替を可能にする。

## 選択理由

`NextIntlClientProvider` はクライアントコンポーネントのため、`locale` と `messages` を React state で管理すれば、ページリロードなしで言語切替できる。`localStorage` でロケール設定を永続化する。

- Web アプリも恩恵を受ける（リロード不要になる）
- editor-core は後方互換（`onLocaleChange` は optional prop）
- VSCode 拡張は変更不要（`hideSettings` でスイッチャー非表示）

## 実装ステップ

### Step 1: `LocaleProvider.tsx` 新規作成

- `'use client'` コンポーネント
- ja.json, en.json を両方 static import（合計 ~30KB、許容範囲）
- `useState` でロケール管理、初期値は `localStorage` → `serverLocale` prop → `'ja'` の順でフォールバック
- `setLocale()` で state 更新 + `localStorage` 保存 + cookie 設定（Web SSR 用）
- `LocaleContext` と `useLocaleSwitch()` hook をエクスポート
- `NextIntlClientProvider` をラップして `locale` と `messages` を state から供給

### Step 2: `EditorSettingsPanel` に `onLocaleChange` prop 追加

- `EditorSettingsPanelProps` に `onLocaleChange?: (locale: string) => void` 追加
- `handleLocaleChange`: `onLocaleChange` があればそれを呼ぶ、なければ既存の cookie+reload（後方互換）

### Step 3: `MarkdownEditorPage` に `onLocaleChange` prop 追加

- `MarkdownEditorPageProps` に `onLocaleChange?: (locale: string) => void` 追加
- `EditorSettingsPanel` に `onLocaleChange` を伝播

### Step 4: `layout.tsx` を `LocaleProvider` に切り替え

- `NextIntlClientProvider` を `LocaleProvider` に置換
- `getMessages()` の呼び出しを削除（LocaleProvider 内で管理）
- `serverLocale={locale}` を渡す

### Step 5: `page.tsx` で `onLocaleChange` を接続

- `useLocaleSwitch` を import
- `MarkdownEditorPage` に `onLocaleChange={setLocale}` を渡す
