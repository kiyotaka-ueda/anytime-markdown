# 7. 保存・エクスポートする

## ファイルの保存

### 名前を付けて保存

1. `Ctrl+Shift+S` を押すか、ツールバーの **名前を付けて保存** ボタンをクリックします。
2. ファイル名と保存先を指定して保存します。
3. Markdown（`.md`）形式で保存されます。

### 上書き保存

ファイルを開いている場合、`Ctrl+S` で同じファイルに上書き保存できます。

File System Access API 非対応のブラウザ（Safari 等）では、上書き保存の代わりにダウンロードが実行されます。

### 自動保存

編集中の内容はブラウザの localStorage に自動保存されます。明示的な保存操作は不要ですが、ファイルとして残すには上記の保存操作が必要です。

## 既存ファイルを開く

1. `Ctrl+O` を押すか、ツールバーの **開く** ボタンをクリックします。
2. ファイル選択ダイアログから `.md` ファイルを選択します。
3. ファイルの内容がエディタに読み込まれます。

## Markdown ファイルのインポート

`.md` ファイルをエディタ領域にドラッグ&ドロップすると、ファイルの内容が読み込まれます。

## PDF エクスポート

設計書を PDF 形式で出力します。印刷用レイアウトや社内配布用に利用します。

1. ツールバーの **PDF エクスポート** ボタンをクリックします。
2. ブラウザの印刷ダイアログが表示されます。
3. 「PDF として保存」を選択して保存します。

PDF エクスポート時の注意事項:
- YAML フロントマターは PDF に含まれません
- Source モードでは PDF エクスポートは利用できません
- 改ページガイドを有効にすると、エディタ上で改ページ位置を事前に確認できます（エディタ設定 → 改ページガイド）

## エンコーディングの変更

ファイルの文字エンコーディングを変更できます。

1. ステータスバーのエンコーディング表示（例: **UTF-8**）をクリックします。
2. 変更先のエンコーディングを選択します。

| エンコーディング | 用途 |
|----------------|------|
| **UTF-8** | 標準（推奨） |
| **Shift_JIS** | 既存の日本語ドキュメントとの互換性 |
| **EUC-JP** | レガシーシステムとの互換性 |

変更前に確認ダイアログが表示されます。エンコーディング変更により文字化けが発生する可能性があるため、事前にバックアップを取ることを推奨します。

## 改行コードの変更

ファイルの改行コードを切り替えできます。

1. ステータスバーの改行コード表示（**LF** または **CRLF**）をクリックします。
2. LF と CRLF が切り替わります。

| 改行コード | 用途 |
|-----------|------|
| **LF** | Unix / macOS / Linux 標準（推奨） |
| **CRLF** | Windows 標準 |

Git での管理を考慮し、通常は **LF** を推奨します。
