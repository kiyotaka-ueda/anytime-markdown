# Anytime Markdown Editor

**AIが生成したドキュメントをコードと一緒に管理 — VS Code ひとつで完結。**

現代の開発ワークフローでは、AIアシスタントが生成した仕様書、設計書、議事録、運用手順書など、大量のMarkdownファイルが生まれます。Anytime Markdown を使えば、**ソースコードとは別のドキュメント管理用Gitリポジトリ**を VS Code 内で直接開き、AIが出力したMarkdownをリッチエディタで確認しながらアプリケーション開発を進められます。

## Anytime Markdown が解決する課題

| 課題 | Anytime Markdown の解決策 |
| --- | --- |
| AIが出力したMarkdownをプレーンテキストで確認するのが大変 | WYSIWYGエディタでダイアグラム・表・数式をライブレンダリング |
| ドキュメントが別リポジトリにあるが、コーディング中に参照したい | マークダウン管理パネルで複数Gitリポジトリを同時に表示 |
| AI生成ドキュメントの差分レビューが面倒 | ブロック単位の差分ハイライト付き比較モード |
| エディタとGit GUIの切り替えで集中が途切れる | Git変更・グラフ・タイムラインをサイドバーに統合 |

## 主な機能

### マークダウン管理パネル — ドキュメントのハブ

ドキュメント専用のGitリポジトリを1つ以上開けます。各リポジトリはトップレベルノードとして表示され、ファイルツリー、ブランチ切替、ファイル操作（作成・名前変更・削除・ドラッグ&ドロップ）が可能。Markdownのみ表示フィルターで視認性を確保。

### Git ワークフロー内蔵

- **変更パネル**: リポジトリごとにステージ済み/未ステージの変更をグループ表示。ステージ・コミット・プッシュまで完結
- **Git グラフ**: カラーアイコン付きのコミット履歴（青=ローカル、赤=リモート済み）
- **タイムライン**: ファイルごとのコミット履歴。コミットを選択して比較パネルに読み込み

### リッチ WYSIWYG 編集

Markdownをリアルタイムプレビュー付きで視覚的に編集。テーブル（Excel/Google Sheetsからの貼り付け対応）、Mermaid/PlantUMLダイアグラム、KaTeX数式、37言語対応のシンタックスハイライト、アドモニション、タスクリスト、脚注などをサポート。

### 比較 & マージモード

ブロック単位の差分ハイライト付き左右比較。任意のファイルや過去のコミットを比較パネルに読み込み可能。全画面テーブル比較ではセル単位の差分ハイライトを表示。

### 3つのエディタモード

| モード | 説明 |
| --- | --- |
| **WYSIWYG** | リッチテキスト編集。書式設定やブロック挿入を視覚的に操作 |
| **ソース** | 生のMarkdownテキストを直接編集 |
| **レビュー** | AI生成コンテンツのレビュー用閲覧専用モード。コメントとチェックボックスのみ操作可能 |

ツールバーのpill型トグルまたは `Ctrl+Alt+S` で切り替え。

## 典型的なワークフロー

1. AIアシスタントが仕様書や設計書をMarkdownで生成
2. **マークダウン管理**パネルでドキュメントリポジトリをクローンまたは開く
3. **レビュー**モードまたは**WYSIWYG**モードで生成内容を確認
4. **比較モード**で以前のバージョンとの差分を確認
5. 編集・コミット・プッシュ — すべてサイドバーパネルから操作
6. VS Code でコーディングを継続。ドキュメントはいつでもアクセス可能

## スラッシュコマンド

エディタで `/` を入力して要素を挿入:

`/heading1`..`/heading3`, `/bulletList`, `/orderedList`, `/taskList`, `/blockquote`, `/codeBlock`, `/table`, `/horizontalRule`, `/mermaid`, `/plantuml`, `/math`, `/html`, `/toc`, `/date`, `/footnote`, `/note`, `/tip`, `/important`, `/warning`, `/caution`, `/comment`

## 拡張機能の設定

| 設定 | 型 | デフォルト | 説明 |
| --- | --- | --- | --- |
| `anytimeMarkdown.fontSize` | number | `0` | フォントサイズ（px）。0 で VS Code デフォルト |
| `anytimeMarkdown.editorMaxWidth` | number | `0` | エディタの最大幅（px）。0 で制限なし |

## 使い方

`.md` / `.markdown` ファイルを開くと、自動的に Anytime Markdown エディタで表示されます。

VS Code 標準のテキストエディタで開きたい場合は、ファイルを右クリック → **「Open With...」** から **「Text Editor」** を選択してください。

## 動作要件

- VS Code 1.109.0 以上

## 既知の問題

- 画像のドラッグ&ドロップは base64 としてファイルに埋め込まれます。大きな画像は Markdown ファイルのサイズが増大するため注意してください。
- TipTap の Markdown ラウンドトリップにより、読み込み時にフォーマット整形が発生する場合があります。初回読み込み時に通知でお知らせします。
